Cấu hình VPN Connection

Cấu hình VPN Connection

  1. Đăng nhập vào AWS Management Console.
  • Tìm EC2
  • Chọn EC2

  1. Truy cập vào trang EC2.
  • Chọn Instances
  • Chọn On-Premises Customer Gateway
  • Chọn Connect

  1. Ở trang Connect to instance.
  • Chọn Session Manager
  • Chọn Connect

  1. Cài đặt OpenSwan.
sudo yum install openswan -y

  1. Cấu hình file /etc/sysctl.conf.
sudo nano /etc/sysctl.conf
  • Chuyển xuống vị trí cuối cùng trong file cấu hình rồi thêm đoạn sau vào cuối tập tin cấu hình.
net.ipv4.ip_forward = 1
net.ipv4.conf.default.rp_filter = 0
net.ipv4.conf.default.accept_source_route = 0

  • Ấn Ctrl+O, Enter, Ctrl+X để lưu file cấu hình.
  1. Sau đó để áp dụng cấu hình này, chạy lệnh:
sudo sysctl -p

  1. Mở file cấu hình mà bạn đã tải xuống từ phần trước.

  1. Tiếp theo chúng ta sẽ cấu hình file /etc/ipsec.d/aws.conf.
sudo vi /etc/ipsec.d/aws.conf
  • Ấn phím i để tiến hành chỉnh sửa file.
  • Sao chép và dán conn Tunnel1 và conn Tunnel2 từ hướng dẫn Bước 4 của file cấu hình đã tải xuống.
  • Đảm bảo bạn thực hiện các thay đổi sau:
    • Đối với Amazon Linux thì chúng ta sẽ xóa dòng auth=esp trong file cấu hình gốc.
    • Thay thế aes128-sha1;modp1024 trên dòng phase2alg bằng aes_gcm.
    • Thay thế aes128-sha1;modp1024 trên dòng ike bằng aes256-sha2_256;dh14.
    • Vì chúng ta chỉ có 1 public IP address cho Customer Gateway nên sẽ cần thêm cấu hình overlapip=yes.
    • leftid: IP Public Address phía Onpremise (ở đây chính là IP public của On-Premises Customer Gateway).
    • right: IP Public Address phía AWS VPN Tunnel.
    • leftsubnet: CIDR của Mạng phía Local.
    • rightsubnet: CIDR của Mạng phía Private Subnet trên AWS.
conn Tunnel1
	authby=secret
	auto=start
	left=%defaultroute
	leftid=13.229.122.160
	right=18.136.181.114
	type=tunnel
	ikelifetime=8h
	keylife=1h
	phase2alg=aes_gcm
	ike=aes256-sha2_256;dh14
	keyingtries=%forever
	keyexchange=ike
	leftsubnet=<LOCAL NETWORK>
	rightsubnet=<REMOTE NETWORK>
	dpddelay=10
	dpdtimeout=30
	dpdaction=restart_by_peer
	overlapip=yes
	
conn Tunnel2
	authby=secret
	auto=start
	left=%defaultroute
	leftid=13.229.122.160
	right=18.136.181.114
	type=tunnel
	ikelifetime=8h
	keylife=1h
	phase2alg=aes_gcm
	ike=aes256-sha2_256;dh14
	keyingtries=%forever
	keyexchange=ike
	leftsubnet=<LOCAL NETWORK>
	rightsubnet=<REMOTE NETWORK>
	dpddelay=10
	dpdtimeout=30
	dpdaction=restart_by_peer
	overlapip=yes

  • Ấn phím ESC và tổ hợp :x để lưu file cấu hình.
  1. Cấu hình file /etc/ipsec.d/aws.secrets.
sudo vi /etc/ipsec.d/aws.secrets
  • Ấn phím i để tiến hành chỉnh sửa file.
  • Thêm đoạn sau vào file cấu hình (đoạn cấu hình này ở bước 5 của IPSEC Tunnel #1 và IPSEC Tunnel #2).
13.229.122.160 18.136.181.114: PSK "HHF1TslQzXbeGXVZx_l9SiuSAHg1XHEA"
13.229.122.160 52.77.21.130: PSK "Cjbzt4W65ZVKOENSsvs85pdokmWVcxsP"

  • Ấn phím ESC và tổ hợp :x để lưu file cấu hình.
  1. Enable và start Openswan.
sudo systemctl enable ipsec.service
sudo ipsec start

  • Nếu status tunnel vẫn chưa chạy đúng, sau khi kiểm tra và cập nhật cấu hình bạn sẽ cần chạy lệnh để restart lại service network và IPsec.
sudo service network restart
sudo service ipsec restart
  1. Truy cập giao diện VPC. Chọn Site-to-Site VPN connections bạn sẽ thấy trạng thái của Tunnel là Up, có nghĩa là chúng ta đã cấu hình thành công.

  1. Truy cập giao diện VPC.
  • Chọn Route tables ở thanh bên trái
  • Chọn VPC A Private Route Table
  • Chọn Routes
  • Chọn Edit routes

  1. Chọn Add route.
  • Destination: nhập 10.3.0.0/16 - Target: chọn Transit Gateway - VPC A Attachment
  • Chọn Save changes

  1. Route table đã được cập nhật.

  1. Lặp lại tương tự để thêm Transit Gateway cho route table VPC B Private Route Table.

  1. Lặp lại tương tự để thêm Transit Gateway cho route table VPC C Private Route Table.

  1. Bây giờ chúng ta hãy thử ping từ On-Premises Customer Gateway đến private IP của VPC A Private AZ1 Server và thấy ping thành công.
ping 'Private IPv4 addresses' -c5

  1. Chúng ta cũng có thể thử ping từ On-Premises Customer Gateway đến private IP của VPC B Private AZ1 Server và VPC C Private AZ1 Server hoặc ngược lại.
